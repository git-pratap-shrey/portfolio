\textbf{2. uniAI} — \textit{Syllabus-Aware AI Study Assistant} \\ % [cite: 156]
\textit{GenAI / RAG System} % [cite: 156]
\begin{itemize}
\item \textbf{Multi-Stage RAG Architecture:} Orchestrated an end-to-end pipeline across three isolated ChromaDB collections (syllabus, notes, PYQs) with a modular model registry supporting seamless switching between Gemini, Ollama, and Groq. % [cite: 156, 160]
\item \textbf{Hybrid Routing \& Query Expansion:} Engineered a sub-millisecond, four-tier router (Regex $\rightarrow$ Keyword $\rightarrow$ Embedding $\rightarrow$ LLM fallback) coupled with a three-layer query expansion system to bridge the gap between student terminology and academic text. % [cite: 157, 158]
\item \textbf{VLM-Powered Data Pipeline:} Automated high-fidelity extraction of complex exam papers using Qwen3-VL, ensuring semantic sectioning and structured JSON outputs under rate-limited environments. % [cite: 157]
\item \textbf{Precision Retrieval \& DevOps:} Prevented LLM hallucinations by coupling GPU-accelerated cross-encoder reranking (Qwen3) with strict out-of-syllabus gating, validated by a comprehensive CI/CD testing suite using GitHub Actions and pytest. % [cite: 158, 159]
\end{itemize}
\textbf{GitHub:} \href{https://github.com/git-pratap-shrey/uniAI}{github.com/git-pratap-shrey/uniAI} % [cite: 160]
